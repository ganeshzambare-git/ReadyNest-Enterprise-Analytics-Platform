# Power BI & Enterprise Deployment Guide

This document outlines the deployment strategy for shifting the analytical workloads generated by the Python/Streamlit backend into a managed Enterprise Visualization Server like **Power BI Service** or **Tableau Server**.

## 1. Cloud Publishing Strategy

To transition from a local Streamlit dashboard to Power BI/Tableau:

1. **Database Export:** Use the `Export` tab in the Streamlit Data Loading module (or the automated `sql_migration.py` script) to push the mathematically cleaned data and engineered features (CLV, Churn Risk) to the central PostgreSQL/MySQL database.
2. **DirectQuery vs Import:**
   - **Import Mode (Recommended):** Set Power BI to import data daily at 3:00 AM. This guarantees lightning-fast dashboard performance for executives.
   - **DirectQuery:** Use this only if real-time (minute-by-minute) sales updates are strictly required.

## 2. Power BI Workspace Management

Deploy the solution using a structured Workspace architecture:

- **Workspace Dev:** (`WN_ReadyNest_Dev`) - Where analysts build and test new DAX measures and visuals.
- **Workspace UAT:** (`WN_ReadyNest_UAT`) - Where business users test the A/B testing frameworks and validate predictive models.
- **Workspace Prod:** (`WN_ReadyNest_Prod`) - The immutable production environment. Only the automated deployment pipeline can push changes here.

## 3. Row-Level Security (RLS)

To ensure data security, implement RLS in the Power BI Semantic Model based on the `Region` column:

- **Global Execs:** Can view all regions.
- **Regional Managers (e.g., West Region):** Filter context strictly locked to `Region = 'West'`. They cannot see Eastern sales data.

### DAX Implementation:
Create a SQL security mapping table, then apply the following DAX formula to your core Fact Table in Power BI Desktop:
```dax
[Region] = LOOKUPVALUE(SecurityTable[Region], SecurityTable[Email], USERPRINCIPALNAME())
```

## 4. Data Governance & Metadata Framework

> [!IMPORTANT]
> A central tenet of enterprise analytics is trusting the data. The Business Glossary must be maintained in the BI Portal. Ensure the Power BI Semantic Model is marked as "Certified" to indicate it is the single source of truth containing the XGBoost ML predictions.

### Data Lineage
Data Lineage tracking should be enabled within Power BI Service to trace metrics backward from the Dashboard → Dataset → Dataflow → SQL Database. This ensures rapid debugging if a predictive model outputs anomalous metrics.
